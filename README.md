# HR Management App

A full-stack HR management system for managing employees, departments, positions, leave requests, approvals, and role-based access.

Live app: https://hr-management-app-dusky.vercel.app

## Overview

The app supports three main roles:

- Admin: manages employees, departments, positions, approvals, and leave requests.
- HR Manager: manages employee records and reviews leave requests.
- Employee: views their own profile, submits leave requests, and tracks leave history.

Employee accounts created by HR are approved immediately. A temporary password is generated by the backend and sent to the employee by email. Public signups are created as pending accounts and require approval before login.

## Features

- Authentication with access and refresh tokens
- Role-based frontend and backend access control
- Employee list, profile details, creation, editing, approval, and deactivation
- Department and position management
- Hard-coded permission sets by role/access level
- Leave request creation, history, approval, and rejection
- Dashboard summaries for management and employee users
- Email notifications for employee creation, password changes, deactivation, and leave updates
- Swagger API documentation

## Backend Setup

From the project root:

```bash
cd backend
npm install
```

Create `backend/.env`:

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DATABASE"
PORT=5000
FRONTEND_URL="http://localhost:3000"

JWT_ACCESS_SECRET="replace-with-a-long-random-secret"
JWT_REFRESH_SECRET="replace-with-another-long-random-secret"
JWT_ACCESS_EXPIRE="15m"
JWT_REFRESH_EXPIRE="7d"

EMAIL_SERVICE="gmail"
EMAIL_USER="your-email@example.com"
EMAIL_PASS="your-email-app-password"
EMAIL_FROM="your-email@example.com"
APP_NAME="HR Management System"

ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="AdminPass123!"
```

Run Prisma and seed the first admin user:

```bash
npm run prisma:generate
npm run prisma:migrate
npm run seed
```

Start the backend:

```bash
npm run dev
```

The API runs at:

```text
http://localhost:5000
```

API docs are available at:

```text
http://localhost:5000/api/docs
```

## Frontend Setup

## Production Notes

- Set `VITE_API_BASE_URL` in Vercel to the deployed backend API URL ending in `/api`.
- Set `FRONTEND_URL` in the backend environment to `https://hr-management-app-dusky.vercel.app`.
- Keep JWT secrets and email credentials private.
- Run `npm run prisma:deploy` on the backend when deploying migrations to production.

## Login Flow

1. Seed an admin user with `ADMIN_EMAIL` and `ADMIN_PASSWORD`.
2. Sign in as admin.
3. Create departments and positions.
4. Add employees from the Employees page.
5. The backend emails each new employee their temporary password.
6. Employees sign in and can access their profile and leave requests.
